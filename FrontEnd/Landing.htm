1 Click Demo!



    This is an automated Demo service, sole purpose, 1 click Nutanix Demos!
    This services runs in a Nutanix datacenter using Windows Powershell
    Once the task is created the system will build several demos completely installed and ready to go.
    The source should be a blanc Foundation PE cluster, with no logins since its foundation.
    This can be a physical block in our office, or a hosted POC, Anything reachable in our network.
    Please read the instructions below to get started! Keep in mind this system is not final yet.
  1. Forward your Hosted POC email (Foundation Finished) to michell.grauwmans@nutanix.com
    Wait for the reply and login here.
    You can check the Queued page if you recieved a validation error.
    Or

    Slackbot in #Foundation_addon Channel (Beta!)
    Say something with the word "build" or "start"

    Or

    The Queued page on this website
  2. Please do not login to the cluster until you have recieved confirmation this process is completed.)
    Password set/reset, register, etc, will be done by this service.
  3. This process takes about 65 minutes depending on the options selected, the GEO location of the cluster and other factors.
    There is a lot of redundancy in this system, its as failure resistant as feasible.
    This service builds with several threads in parallel, yet the core takes this long to run.
    Image download, PC Download, PC install and LCM updates are the main reasons.
    This system uses a message-bus/queuing system, at start keep track of your email, incase the request is not validated.
    You will recieve a queued and started mail, read all the Spec details, IPs, Domain Name etc there.
    Once running, you should recieve a mail after ~65 Minutes indicating a success.
    The auto abort on failure is not implemented completly yet.
    Ping me for questions or read the log.( Running or History Tabs)
    Once completed login (PE/PC) with first.middle.last@pocname.nutanix.local using the password specified by you.
    For Applications login names can differ, but all share the same password.(with the exception of AHV hosts)
  4. 1 Click Demo requires several images to be pushed.
    Depending on the PE internet Bandwidth, this can take several hours to complete. HPOC is fast and takes only minutes per image.
    If this is not a hosted POC, you can specify an EU/US region for downloads, using the new entry on the Queued page.
  5. When forwarding the mail, use one of the following parameters to configure the deployment.
    Please use the format -newline-parametername:value no spaces, one parameter per line, the first 25 lines are read.
    In terms of 1 click simplicity, the options below do not need to be specified;
    These settings are all on by default or provide the most logical choice.

    PCmode:1|3
    Default is scale out PC, yet automatically set to 1 for configurations or demos with scaleout limitations like Karbon pre GA
    queue:manual
    Queues the request on the Queued page. This so it can be edited before submit.
    karbon:1|0
    When set to 1, PC mode is also set to 1, PC setting is overruled.
    Karbon is on the backlog for development.(Soon).
    Images are uploaded.
    era:1|0
    Installs ERA end 2 END. 4 Databases, 4 Time machine DBs, 3 Servers
    files:1|0
    Installs Files using AOS its latest available version(old), Including 3 shares, AD Joined.
    iis:1|0
    Installs XPlay Demo on 5.10.x, Windows IIS server Autoscale including loadbalance, using XPLAY REST and Calm!
    End 2 End automated install.
    Simply generate CPU load and go!
    ssp:1|0
    Creates the 3 Projects each mapped to 1 out of 3 customers in AD, Users and Admins.
    lab:1|0
    Installs Workshop color scheme and changes the login banners.
    slackbot:2|1|0
    Gives normal or info logging to a private Slackbot message. Please mute the bot if higher then 1, 0 is default
